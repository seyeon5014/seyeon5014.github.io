1.3 프레임워크 및 라이브러리 설명

1.3.1 Bootstrap 4

Bootstrap은 반응형 웹 디자인을 쉽게 구현할 수 있게 해주는 CSS 프레임워크입니다. 기본적으로 미리 정의된 CSS 클래스들을 HTML에 적용하기만 하면 데스크톱, 태블릿, 모바일 등 다양한 화면 크기에 자동으로 맞춰지는 웹사이트를 만들 수 있습니다.

본 프로젝트에서 Bootstrap을 사용한 주요 기능:

레이아웃 시스템: Bootstrap의 12칼럼 그리드 시스템을 활용하여 화면을 12등분하고, 각 요소가 몇 칼럼을 차지할지 지정할 수 있습니다. 예를 들어 col-md-6이라는 클래스를 사용하면 중간 크기 화면에서 6칼럼을 차지하는 요소를 쉽게 만들 수 있습니다.

반응형 이미지: img-fluid 클래스를 이미지에 추가하면 부모 요소의 크기에 따라 자동으로 이미지 크기가 조정됩니다. 작은 모바일 화면에서는 작아지고, 큰 데스크톱 화면에서는 커집니다.

컨테이너: container 클래스로 감싼 요소들은 화면 크기에 맞춰 자동으로 좌우 여백이 조정되어 콘텐츠가 너무 가장자리까지 늘어나지 않습니다.

내비게이션 바: navbar 클래스를 이용하면 전문적인 네비게이션 메뉴를 빠르게 구축할 수 있으며, 모바일에서는 햄버거 메뉴로 자동 변환됩니다.

1.3.2 Typed.js

Typed.js는 텍스트를 누군가 실시간으로 타이핑하는 듯한 효과를 만들어주는 자바스크립트 라이브러리입니다. 
웹페이지 처음 접속시 i'm ---- 뒤에 적히는 단어에 사용한 기능 입니다.


헤더 타이핑 애니메이션: 포트폴리오의 제목 영역에 "I'm Coder" 다음 "I'm Guitarist"를 마치 실제로 누군가 타이핑하듯 자동으로 표시하고, 다시 지웠다가 다시 타이핑하는 무한 반복 효과를 만들었습니다.

1.3.3 jQuery

jQuery는 자바스크립트를 더 간단하고 효율적으로 사용할 수 있게 해주는 자바스크립트 라이브러리입니다. 복잡한 자바스크립트 코드를 간결하게 작성할 수 있으며, 모든 웹 브라우저에서 동일하게 작동합니다.

jQuery의 기능:

DOM 선택 및 조작: jQuery의 $(선택자) 문법으로 HTML 요소를 쉽게 찾고 그 요소의 속성을 변경할 수 있습니다. 예를 들어 네비게이션 링크를 클릭할 때의 동작을 처리하는 방식을 비교하면:

순수 JavaScript 방식:
```javascript
var navLinks = document.querySelectorAll('nav a');
for (var i = 0; i < navLinks.length; i++) {
  navLinks[i].addEventListener('click', function(event) {
    event.preventDefault();
    var targetId = this.getAttribute('href');
    var targetElement = document.getElementById(targetId);
    if (targetElement) {
      targetElement.scrollIntoView({ behavior: 'smooth' });
    }
  });
}
```

jQuery를 이용한 방식:
```javascript
$('nav a').click(function(event) {
  event.preventDefault();
  var targetId = $(this).attr('href');
  $(targetId).stop().animate({ scrollTop: 0 }, 1000);
});
```

보다시피 jQuery는 코드가 훨씬 짧고 읽기 쉽습니다. $('nav a')로 모든 네비게이션 링크를 한 번에 선택하고, .click() 함수로 클릭 이벤트를 바로 처리할 수 있습니다. 순수 JavaScript에서는 querySelectorAll()로 선택한 후 for 루프를 돌면서 각각 addEventListener를 추가해야 합니다.

부드러운 스크롤 애니메이션: 네비게이션 메뉴를 클릭하면 페이지가 해당 섹션으로 부드럽게 스크롤되는 효과를 jQuery의 animate() 함수로 쉽게 구현합니다. jQuery의 .animate()는 복잡한 수학 계산 없이도 자동으로 부드러운 애니메이션을 만들어줍니다.

요소의 보이기/숨기기: jQuery의 show(), hide(), toggle() 함수를 이용하여 모바일 메뉴의 오픈/클로즈 기능을 간단하게 구현할 수 있습니다. 예를 들어 $('mobile-menu').toggle()이라는 한 줄의 코드로 요소를 보이거나 숨길 수 있으며, 순수 JavaScript에서는 style.display 속성을 일일이 제어해야 합니다.

3. 핵심 기능 구현

3.1 반응형 네비게이션 시스템

구현 내용:
데스크톱과 모바일 환경을 구분하는 반응형 네비게이션 메뉴를 개발하였습니다. CSS Media Query를 활용하여 화면 크기별로 최적화된 레이아웃을 제공합니다.

핵심 코드 (style.css):

```css
.nav-menu {
  margin-top: 60px;        /* 헤더 타이틀과의 간격을 확보하여 여백을 만듬 */
  width: 100%;             /* 전체 너비를 차지하여 중앙 정렬의 기준점 설정 */
}

.nav-menu ul {
  display: flex;           /* Flexbox 레이아웃 활성화로 가로 배치 구현 */
  margin: 0 auto;          /* 좌우 자동 마진으로 중앙 정렬 */
  padding: 0;              /* 기본 패딩 제거로 정확한 위치 제어 */
  list-style: none;        /* 불릿 포인트 제거하여 깔끔한 메뉴 구현 */
  justify-content: center; /* Flex 아이템들을 중앙에 배치 */
}

.nav-menu a {
  display: block;          /* 블록 레벨 요소로 변경하여 클릭 영역 확대 */
  position: relative;      /* 가상 요소(::before)의 기준점 설정 */
  color: #C2B4D6;         /* 메뉴 텍스트 색상 지정 */
  font-size: 19px;        /* 가독성을 고려한 적절한 폰트 크기 */
  font-family: "Poppins", sans-serif;  /* 모던한 느낌의 폰트 적용 */
  font-weight: 400;       /* 일반 두께로 설정하여 과도한 강조 방지 */
}

.nav-menu a:before {
  content: "";                      /* 빈 가상 요소 생성 */
  position: absolute;               /* 절대 위치로 텍스트 하단에 배치 */
  width: 0;                        /* 초기 너비 0으로 설정 (애니메이션 시작점) */
  height: 2px;                     /* 밑줄 두께 설정 */
  bottom: -4px;                    /* 텍스트 아래 4px 위치에 배치 */
  left: 0;                         /* 왼쪽부터 애니메이션 시작 */
  background-color: #C2B4D6;       /* 밑줄 색상 지정 */
  visibility: hidden;              /* 초기 상태에서 숨김 */
  transition: all 0.3s ease-in-out 0s;  /* 0.3초 동안 부드러운 전환 효과 */
}

.nav-menu a:hover:before {
  visibility: visible;     /* 마우스 오버 시 밑줄 표시 */
  width: 25px;            /* 밑줄이 25px 길이로 확장되는 애니메이션 */
}
```

3.3 비디오 모달 시스템

모달(Modal)이란 기존 화면 위에 새로운 창을 띄우는 UI 요소입니다.

구현 내용:
클릭 시 전체 화면 모달로 동영상을 재생하며, 외부 클릭이나 닫기 버튼으로 모달을 닫을 수 있습니다.

HTML 구조:

```html
<div id="videoModal" style="display: none; position: fixed; z-index: 1000; 
     left: 0; top: 0; width: 100%; height: 100%; 
     background-color: rgba(0,0,0,0.8);">
  <!-- display: none으로 초기에는 숨김 상태 -->
  <!-- position: fixed로 스크롤과 무관하게 화면에 고정 -->
  <!-- z-index: 1000으로 다른 요소들 위에 표시 -->
  <!-- 전체 화면을 덮도록 width, height 100% 설정 -->
  <!-- 반투명 검은 배경으로 뒤의 콘텐츠 흐리게 표시 -->
  
  <div style="position: relative; background: #1a1a2e; 
       border-radius: 10px; padding: 40px 20px 20px 20px; 
       max-width: 90%; max-height: 85vh;">
    <!-- position: relative로 내부 절대 위치 요소의 기준점 -->
    <!-- max-width: 90%로 화면 양쪽에 여백 확보 -->
    <!-- max-height: 85vh로 화면 높이의 85% 이내로 제한 -->
    <!-- padding-top: 40px로 닫기 버튼 공간 확보 -->
    
    <span onclick="closeVideoModal()" 
          style="position: absolute; right: 15px; top: 10px; 
          font-size: 30px; cursor: pointer;">&times;</span>
    <!-- position: absolute로 컨테이너 우측 상단에 고정 배치 -->
    <!-- cursor: pointer로 클릭 가능함을 시각적으로 표시 -->
    <!-- &times;는 × 기호를 나타내는 HTML 엔티티 -->
    
    <video id="modalVideo" width="100%" height="auto" 
           controls autoplay style="max-height: 80vh;">
      <!-- controls로 재생/일시정지 등 컨트롤러 표시 -->
      <!-- autoplay로 모달 열릴 때 자동 재생 -->
      <!-- max-height: 80vh로 세로 영상도 스크롤 없이 표시 -->
      <source src="" type="video/mp4">
    </video>
  </div>
</div>
```

JavaScript 로직:

```javascript
function openVideoModal(videoSrc) {
  const modal = document.getElementById('videoModal');  // 모달 요소 선택
  const video = document.getElementById('modalVideo');  // 비디오 요소 선택
  video.src = videoSrc;    // 클릭한 비디오의 경로를 소스로 설정
  modal.style.display = 'flex';  // 모달을 화면에 표시 (flex로 중앙 정렬)
}

function closeVideoModal() {
  const modal = document.getElementById('videoModal');  // 모달 요소 선택
  const video = document.getElementById('modalVideo');  // 비디오 요소 선택
  video.pause();           // 재생 중인 비디오 일시정지
  video.src = '';         // 소스 초기화로 메모리 절약
  modal.style.display = 'none';  // 모달 숨김
}

// 모달 외부 클릭 시 닫기 기능
window.onclick = function(event) {
  const videoModal = document.getElementById('videoModal');  // 모달 요소 선택
  if (event.target == videoModal) {  // 클릭 대상이 모달 배경인지 확인
    closeVideoModal();               // 배경 클릭 시 모달 닫기
  }
}
```

4. 주요 기술적 구현 사항

4.1 Bootstrap 프레임워크를 활용한 반응형 디자인

본 프로젝트는 다양한 디바이스 환경에서 최적의 사용자 경험을 제공하기 위해 Bootstrap 프레임워크의 반응형 기능을 활용하였습니다.

Bootstrap의 그리드 시스템 활용: Bootstrap의 12칼럼 그리드 시스템을 이용하여 화면을 논리적으로 분할합니다. 예를 들어 비디오 섹션에서는 col-lg-3(큰 화면에서 3칼럼), col-md-6(중간 화면에서 6칼럼) 클래스를 사용하여 자동으로 화면 크기에 맞는 레이아웃을 만듭니다.CSS Media Query를 일일이 작성할 필요 없이 Bootstrap이 제공하는 미리 정의된 클래스만 사용하면 반응형 웹사이트를 빠르게 완성할 수 있습니다.

이미지 갤러리의 자동 조정: 본 프로젝트의 이미지 갤러리는 Bootstrap의 컨테이너와 CSS Grid를 결합하여 구현하였습니다. 화면 너비에 따라 한 줄에 표시되는 이미지 개수가 자동으로 조정되며, 모바일에서는 1개, 태블릿에서는 2~3개, 데스크톱에서는 4~5개의 이미지가 한 줄에 표시됩니다.

Bootstrap의 버튼 및 폼 스타일: Bootstrap이 제공하는 미리 정의된 버튼과 폼 요소 스타일을 사용하여 일관되고 전문적인 모양의 UI를 빠르게 구현할 수 있습니다. 추가적인 CSS 작성 없이도 클릭하기 좋은 크기의 버튼과 깔끔한 입력 폼을 만들 수 있습니다.

CSS Media Query 활용: Bootstrap의 반응형 기능을 넘어서, 커스텀 Media Query를 작성하여 992px 이하에서는 데스크톱 네비게이션을 숨기고 모바일 전용 햄버거 메뉴를 표시합니다. 이는 작은 화면에서 네비게이션이 차지하는 공간을 줄여 콘텐츠에 더 많은 영역을 할당합니다.

4.2 CSS3 레이아웃 

Flexbox를 통한 일차원 레이아웃: 네비게이션 메뉴는 display: flex와 justify-content: center를 사용하여 메뉴 항목들을 중앙에 정렬합니다. Flexbox는 한 줄 또는 한 열로 요소들을 정렬하는 데 최적화되어 있으며, align-items와 justify-content 속성으로 요소들을 손쉽게 정렬할 수 있습니다.

CSS Grid를 통한 이차원 레이아웃: 이미지 갤러리와 비디오 섹션은 display: grid와 grid-template-columns: repeat(auto-fit, minmax(250px, 1fr))를 사용하여 구현하였습니다. 이 기법은 화면 너비에 따라 열의 개수가 자동으로 조정되므로, Media Query를 여러 번 작성할 필요가 없습니다.

Box Shadow와 Border Radius: 모든 카드 요소(비디오, 이미지, 연락처 정보 박스)에 box-shadow를 적용하여 입체적인 그림자 효과를 주었고, border-radius로 모서리를 둥글게 처리하여 현대적인 디자인을 표현합니다.

CSS Transition과 Transform: 네비게이션 메뉴 항목에 마우스를 올리면 밑줄이 부드럽게 나타나는 애니메이션은 transition: all 0.3s ease-in-out을 사용합니다.

4.3 Typed.js를 활용한 동적 텍스트 애니메이션

타이핑 애니메이션의 원리: Typed.js는 문자열 배열을 받아 첫 번째 문자열을 한 글자씩 타이핑하고, 완료 후 지우기를 반복합니다. typeSpeed 속성으로 글자 타이핑 속도를 조정하고, backSpeed로 지우기 속도를 제어합니다. loop: true 옵션으로 이 과정을 무한 반복하도록 설정합니다.


4.4 jQuery를 통한 DOM 조작 및 이벤트 처리

jQuery는 자바스크립트의 복잡한 문법을 간결하게 만들어주는 라이브러리

DOM 선택의 간결성: 순수 JavaScript에서는 document.getElementById('elementId')로 5개 단어를 작성해야 하지만, jQuery에서는 $('#elementId')로 단 3글자 만에 같은 작업을 수행합니다.

모바일 메뉴 토글: 모바일에서 햄버거 메뉴 아이콘을 클릭하면 메뉴가 슬라이드되면서 나타나는 효과는 jQuery의 slideToggle() 또는 fadeToggle() 함수로 구현합니다.

4.5 Vanilla JavaScript를 통한 모달 시스템 구현

모달 시스템의 필요성: 이미지와 비디오를 전체 화면으로 확대 표시하려면 원본 크기를 보존하면서도 사용자가 쉽게 종료할 수 있는 인터페이스가 필요합니다. 이를 위해 Vanilla JavaScript(순수 자바스크립트)로 커스텀 모달을 직접 구현하였습니다.

DOM 조작의 기초: JavaScript의 document.getElementById()로 모달 요소를 선택한 후, style.display 속성을 변경하여 모달을 보이거나 숨깁니다. 이는 라이브러리 없이 자바스크립트의 기본 기능만으로 복잡한 상호작용을 만들 수 있음을 보여줍니다.

이벤트 버블링의 활용: 모달 외부를 클릭하면 자동으로 모달이 닫히는 기능은 이벤트 버블링 개념을 활용합니다. 전체 화면의 클릭 이벤트를 감지하고, 클릭 대상이 모달 배경인지 확인하여 배경 클릭 시에만 닫도록 구현합니다.

4.6 Bootstrap 그리드와 CSS Grid의 결합

효율적인 레이아웃 구성: Bootstrap의 컨테이너와 행/열 개념으로 전체 페이지의 큰 레이아웃을 잡은 후, 각 섹션 내에서는 CSS Grid로 세부 레이아웃(이미지 갤러리, 비디오 그리드)을 구성합니다. 이는 두 가지 방식의 장점을 모두 활용하는 방식입니다.

불필요한 라이브러리 제거: 초반에 사용했던 owl.carousel(이미지 슬라이더), isotope(그리드 필터링) 같은 무거운 라이브러리는 제거하고 CSS Grid만으로 유사한 기능을 구현함으로써 페이지 로딩 속도를 개선하였습니다.

6. 문제 해결 과정

6.1 About 섹션 정렬 문제

문제: Bootstrap의 Grid 시스템과 커스텀 CSS가 충돌하여 About 섹션의 콘텐츠가 중앙 정렬되지 않음.

해결: !important 플래그를 사용하여 Bootstrap의 기본 스타일을 오버라이드. text-align: center, margin: 0 auto 등에 !important 추가하여 우선순위 상향.

6.2 모바일 메뉴 색상 불일치

문제: 색상 테마 변경 시 데스크톱 네비게이션은 새 색상이 적용되나 모바일 메뉴는 이전 색상 유지.

해결: 데스크톱 네비게이션(.nav-menu)과 모바일 네비게이션(.mobile-nav)의 스타일을 각각 지정. 색상 변경 시 두 가지 모두 수정 필요.

6.3 비디오 모달 스크롤 문제

문제: 세로 영상 재생 시 모달 내부에 스크롤이 발생하여 사용자 경험 저하.

해결: vh 단위를 활용하여 높이 제한. 비디오에 max-height: 80vh, 모달 컨테이너에 max-height: 85vh 적용.
